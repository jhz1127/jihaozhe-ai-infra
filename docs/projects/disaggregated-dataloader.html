分离式 DataLoader

控制面 / 数据面分离，让数据供给与训练计算真正流水并行。

京东实习项目 · PyTorch / Ray / RDMA / GPUDirect RDMA

问题与职责

PyTorch 的 pin_memory_thread 是单线程路径，面对高列数训练数据和跨节点供给时容易成为瓶颈；简单预取也无法消解 shard 不均。我独立设计 M×N Producer/Consumer 架构及其控制面、数据面。

关键设计

  • 中央 Dispatcher 按消费者需求派发 FatPtr 与 ack，避免静态 shard 切分导致的慢 rank。
  • 数据面绕开 CPU staging，直接以 RDMA READ 把数据送至 GPU HBM。
  • 让训练主线程计算与下一 batch 拉取重叠，形成端到端 pipeline。
