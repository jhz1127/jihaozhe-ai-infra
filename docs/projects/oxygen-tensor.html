OxygenTensor 跨节点 Tensor 通信框架

存储管理与通信调度双引擎，为分布式训练提供跨节点 Tensor 传输底座。

京东实习项目 · Python / C++ / CUDA VMM / GPUDirect RDMA

项目背景

大模型训练需要高频跨节点 Tensor 传输。通用 CPU staging 路径会增加拷贝和同步成本，难以充分利用 NIC 与 GPU 的高速数据通路。

架构图

FatPtr / MetaService

远程 Tensor 句柄与元数据寻址

TransferEngine

Mooncake / SDMA 后端抽象与调度

传输路径

GPUDirect RDMA、HBM staging、SHM

工作内容

  • 参与分析 FatPtr、MetaService、TransferEngine 分层，理解存储管理和通信调度的职责边界。
  • 打通 CUDA VMM 的 GPU 虚拟内存映射到 RDMA 可访问物理显存的路径。
  • 梳理并验证 GPUDirect RDMA、HBM staging、SHM 多路径，支持不同拓扑下的可靠传输。

工程价值

该框架将远程 Tensor 的句柄、寻址和数据传输解耦，减少 CPU 参与的数据拷贝，为训练侧提供高吞吐、可扩展的跨节点通信基础能力。
