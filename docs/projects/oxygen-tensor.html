OxygenTensor 通信底座

以统一的远程 Tensor 句柄和多后端路径选择，为搜推训练提供跨进程、跨节点的数据传输能力。

京东实习项目 · C++ / CUDA VMM / RDMA / SHM

背景与职责

分布式训练中的 Tensor 需要在不同进程、GPU 与节点间流动，但不同硬件拓扑适合不同路径。项目希望让上层只面对逻辑 Tensor，把内存位置、注册信息、传输通道和资源回收收敛到基础设施中。我参与存储管理与通信调度的实现与排障，重点覆盖 GPU 内存映射和 RDMA 直通链路。

分层架构

上层 API 不感知物理位置；TransferEngine 根据 FatPtr 和拓扑选择实际传输通路。

关键实现与取舍

  • FatPtr：将地址、rkey、偏移和长度封装为远程句柄，统一本地与远端内存的描述方式。
  • CUDA VMM + GPUDirect：通过虚拟地址映射、内存注册与 NIC DMA，使网络可以直接访问 GPU HBM，减少 CPU staging。
  • 路径降级：优先选择 RDMA / P2P；面对物理页不连续、能力不匹配等条件时回退至 SHM，保证正确性优先。
  • 排障顺序：按 MR 注册、QP 建连、CQ 完成事件逐层定位，避免把链路问题误判成上层 Tensor 生命周期问题。

工程价值

统一句柄跨进程、跨节点定位
多路径按拓扑自动选择
HBM 直通减少中间拷贝

该底座将复杂的内存与通信细节收敛到系统层，为数据供给、训练并行和后续硬件适配提供复用能力。
