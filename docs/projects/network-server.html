高并发网络服务器

基于 C++17、epoll 和 Reactor 模式实现可扩展的事件驱动网络服务框架。

个人项目 · C++17 / epoll / Reactor / 线程池

设计目标

网络服务需要把连接管理、I/O 复用与业务执行隔离：主线程快速接入连接，I/O 线程专注事件循环，耗时业务不阻塞 Reactor。我实现了 Main-Sub Reactor 架构、连接生命周期管理与可增长 Buffer，重点训练对并发边界和资源所有权的处理能力。

架构图

graph LR
  Client["Client"] --> Main["Main Reactor<br/>accept 新连接"]
  Main --> Sub["Sub Reactor × N<br/>EventLoop"]
  Sub --> Channel["Channel<br/>epoll 事件分发"]
  Channel --> Conn["TcpConnection<br/>读写状态 / Buffer"]
  Conn --> Pool["线程池<br/>业务回调"]
  Pool --> Conn

I/O 事件留在 Sub Reactor 中处理，业务逻辑进入线程池，避免单个慢任务阻塞连接循环。

核心实现

  • Main-Sub Reactor：Main Reactor 只处理 accept，按轮询或负载将连接分派给多个 Sub Reactor，减少新连接与读写事件互相干扰。
  • Channel 抽象：将 fd、关注事件与回调绑定，统一由 EventLoop 驱动 epoll 注册、修改和删除。
  • Buffer：使用可增长读写缓冲与分散读，减少小包场景的频繁内存移动和系统调用。
  • 生命周期：用智能指针管理 TcpConnection 与回调捕获，显式处理连接关闭、延迟任务和跨线程投递。

工程收获

该项目形成了从 socket、epoll 到事件循环、线程池和连接生命周期的一套完整实现路径，也是后续理解高性能数据通路与并发系统的基础。
