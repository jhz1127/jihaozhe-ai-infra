Token Budget Batching 动态组批

以 token 总量而非样本数量组织 batch，消除生成式搜推中变长行为序列造成的训练拖尾。

京东实习项目 · PyTorch / Qwen3 / B200 / BF16

背景与职责

用户行为序列长度差异显著，固定 batch_size 会让同样包含 6 条样本的 batch 产生数量级不同的计算量。分布式同步训练中，最慢 rank 决定全局 step 时长，长尾 batch 直接造成设备空转。我独立设计 token budget 动态组批和向量化切分路径，并完成不同预算参数下的性能验证。

组批策略

  subgraph After["After · token_budget=8192"]
    V2["变长样本流"] --> ACC["Token Budget Accumulator<br/>逐样本累计 token<br/>接近预算即产出"]
    ACC --> AB["各 batch token 接近恒定<br/>余额 carry-over"]
    AB --> AL["rank 负载趋稳<br/>提升 MFU 与吞吐"]
  end

动态组批控制每批计算量，而非机械地控制样本数量。

实现要点

  • 以 token budget 为硬约束，处理刚好满、单样本超预算、空序列和结余跨 batch 等边界。
  • 采用 prefix-sum 与二分定位批次边界，替代逐条 Python 循环；切分复杂度从线性扫描降为适合向量化执行的路径。
  • 结合预扫描 token 信息或数据元信息估算开销，让组批决策发生在训练前，避免主循环额外等待。
  • 将 checkpoint 状态扩展为文件、行号与 packing offset，保证恢复后样本消费顺序可追踪。
