Token Budget Batching 动态组批

以 token 总量而非样本数量组织 batch，消除变长序列的训练拖尾。

京东实习项目 · PyTorch / Qwen3 / B200 / ZeRO-1 / BF16

问题与职责

LLM 训练样本长度差异很大，固定 batch_size 下长文本 step 的耗时可达短文本的 3—5 倍。每个 rank 的 token 负载不均，会造成同步等待和 GPU 空转。我独立设计并实现 token budget 动态组批和向量化切分路径。

核心链路

输入样本

变长 token 序列

→
Token Budget

prefix-sum 向量化切分

→
训练 Step

各 rank 负载趋于恒定

关键实现

  • 按 max_tokens 约束累计样本，而非固定样本数；通过 prefix-sum 路径降低切分复杂度。
  • 结合动态 batch 负载均衡，将长尾样本从固定 batch 的拖尾变为可控的 token 预算。
  • 与 FlashDecoding / Continuous Batching 的核心原则一致：减少无效计算与等待。
